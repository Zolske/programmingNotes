Forms and Events

Forms are used to transmit data to the webserver. JavaScript can validate the user input before submitting the form. The user triggers events when interacting with the website. The "eventhandler" is the connection to the JavaScript code.

example of an simple form and events

  1. a reference to the object in the DOM needs to be created
    variable referring to object document-object method id of the html element
    const VARIABLE_OBJECT = document.getElementById ('ID_HTML') ;

  2. the "referenced object" needs to be connected to an event (event listener)
    • the event type needs to be set,
    • a anonymous or named function connected to the event
    assigning an event to a variable containing a referenced object
    variable referring to object element-object method (event type, function to be executed)
    const VARIABLE_OBJECT . addEventListener (click, function) ;

    referencing an object and assigning an event (takes only anony. functions, which itself can call a named function)
    document-object method id of the html element element-object method (event type, function to be executed)
    document.getElementById ('ID_HTML') . addEventListener (click, anonymous function) ;

  3. The location of the "inputs value memory" can be assigned (referenced) to a variable. Any changes to the variable, change the value saved at its referenced memory.
    variable holding value variable referring to object keyword
    const VARIABLE_VALUE = VARIABLE_OBJECT . value ;

    assigning (referencing) the value of an input field directly to a variable
    variable holding value document-object method id of the html element keyword
    const VARIABLE_VALUE = document.getElementById ('ID_HTML') . value ;

  4. Assigning a new value to a referencing variable, overwrites also the value of the object in the DOM
    variable referring to object keyword data to overwrite value
    VARIABLE_OBJECT . value = "STRING" ;
  • preventing an default event (see "required fields and control")
    variable event method
    VARIABLE_EVENT . preventDefault() ;

JavaScript code 
<script>
    function clicked() {                                        //the function clicked is defined
        const inputField = document.getElementById("idInput");  //the html id "idInput" is referenced -
                                                                //to the variable "inputField"        
        const input = inputField.value;                         //the user input which is saved in "value" is -
                                                                //assigned to the variable input
        alert("Click: " + input);                               //alert message with the value of "input"
        const outputField = document.getElementById("idOutput"); //the html id "idOutput" is referenced
                                                                //to the variable "outputField"
        outputField.value = "Click: " + input;                  //the "outputField.value" gets the string "Click"
                                                                //and the content of the variable "outputField"    
    const you = document.getElementById("idButton");    //the html id "idButton" is referenced to the variable "you"
    you.addEventListener("click", clicked);             //"you" is connected to the event "click" and the function
</script>                                               //"clicked" is set to execute when event "click" happens

code explanation
  1. The "idForm" object is connected to the event "submit" which will trigger a anonymous function function(e){return send(e) which itself calls the named function send(e) .
    It can not call the named function straight away because only anonymous functions are aloud when assigning an id object with an event. The event is passed as parameter e . 
        <script>
        document.getElementById("idForm").addEventListener("submit", function(e){
            return send(e);
        });
    </script>
  2. The vale of the object "idPassword" is saved in the constant pw .
    A if condition is triggered if the length of the variable pw is shorter than 3 characters or longer than 8 characters if (pw.length < 3 || pw.length > 8) .
    If true, an alert box opens with the message "Password must contain 3-8 characters" ,
    the method .preventDefault() prevents the event (submitting the form) e from being executed.
    false is returned, the form is not submitted 
        <script>
        function send(e) {
            const pw = document.getElementById("idPassword").value;
            if (pw.length < 3 || pw.length > 8) {
                alert("Password must contain 3-8 characters");
                e.preventDefault();
                return false;
            }
    // ... code continue ...
