HTML: form element

Wraps around the elements of an form, its attributes determine how and where the form is send to.

<form ATTRIBUTE> <FORM_ELEMENTS> </form>

    ATTRIBUTE
  • method=" "
    is how the data is transmitted, post is more secure.
  • action=" "
    where the the data is send to (backend e.g. .php).

HTML: attributes for "input", "textarea" & "select"

    ATTRIBUTE
  • id="IDENTIFIER"
    CSS & JS can "target" the form-element by its unique id
  • name="NAME"
    the value of the input/selection is referenced by the backend under the given "NAME" after submitting.
  • required="required"
    the form can not be submitted if there is no input/selection (error message).
  • disabled
    user can not change the value of the <input>, <textarea> and <select> element, can be changed with JS.
    not disabled , / disabled ,
  • readonly="readonly"
    user can not change the value of the <input> and <textarea> element, can be changed with JS.
    not readonly / readonly

html: "label for" element

  • <label for="VALUE">DESCRIBING_FORM_ELEMENT</label>
    • connects a description (in browser visible) to the "form_element", so the user knows what the input is for.
    • The value of "for" has to match the "id" of the "form_element" for which the label is for.

example of labeling a "form_element" :

 
<label for="radioButton"><i>top</i></label>
<br><label for="radioButton"><i>left</i></label>
<input type="radio" id="radioButton">
<label for="radioButton"><i>right</i></label>
<br><label for="radioButton"><i>bottom</i></label>

receiving a form

  • Is a html file saved with the extension .php instead of .html .
  • php code is written between <?php and ?></p>
  • echo...; displays output to the user.
  • html elements can be used between " "
  •  .  Is used to concatenate a string together.
  • $_POST["NAME_ATTRIBUTE"] evaluates to the data send from the form, which was entered into the <input> field with the attribute name="NAME_ATTRIBUTE"
  • the function htmlentities() helps to protect from user entering malicious code
<?php
   echo "The user has submitted: " . "<mark>" . htmlentities($_POST["entry"]) . "</mark>" . "<br>" . "Thank you :)";
?></p>
