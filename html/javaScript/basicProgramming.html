Performing Operations on Strings

JavaScript provides properties and methods to perform operations on string values. Technically, only objects can have properties and methods. But in JavaScript primitive data types can act like objects when you refer to them with the property access notation (dot notation).

concatenating two or more strings
+
+=		 You can concatenate or combine two or more strings using the + and += assignment operators. 
var hello = "Hello";
var world = "World";
var greet = hello + " " + world;
document.write(greet); // Prints: Hello World
 
var wish  = "Happy";
    wish += " New Year";
document.write(wish); // Prints: Happy New Year
accessing individual characters from a string
VARIABLE[INDEX]
VARIABLE.charAt(INDEX)		 The index determines the position of the to return letter from the string. 
var str = "Hello World!";
document.write(str.charAt());  // Prints: H
document.write(str.charAt(6)); // Prints: W
document.write(str.charAt(30)); // Prints nothing
document.write(str.charAt(str.length - 1)); // Prints: !

var str = "Hello World!";
document.write(str[0]); // Prints: H
document.write(str[6]); // Prints: W
document.write(str[str.length - 1]); // Prints: !
document.write(str[30]); // Prints: undefined
getting the length of a string
VARIABLE.length Returns the length of the string, which is the number of characters contained in the string (including special characters such as \t or \n). 
var str1 = "This is a paragraph of text.";
document.write(str1.length); // Prints 28
 
var str2 = "This is a \n paragraph of text.";
document.write(str2.length); // Prints 30, because \n is only one character

extracting a substring from a string
VARIABLE.slice(startINDEX, endINDEX) Takes 2 parameters: start index (index at which to begin extraction), and an optional end index (index before which to end extraction). If no end index than the rest of the string is returned.

document.write(str.slice(31)); // Prints: the lazy dog.

the number data type

  • Can be represented in decimal (2.1), hexadecimal (0xff) or octal notation.
  • JavaScript does not treat integer (2) and floating-point (3.14) numbers differently.
  • All numbers in JavaScript are represented as floating-point numbers.
  • exponential notation: 4.25e+6 = 4250000 and 4.25e-6 = 0.00000425
  • The underscore _ can be used to visually separate long numbers 1000 = 1_000.

addition on numbers and strings

  • If a sting is part of an addition, than the numbers are converted to strings.

// Adding a string with a string, the result will be string concatenation

// Adding a number with a string, the result will be string concatenation

// Adding a string with a number, the result will be string concatenation

  • Operations a solved from left to right, two or more numbers are added before if a string is coming after.
    var x = 10;

var z = "30";

Avoiding Precision Problems (representation error)

  • JavaScript and many other languages uses binary (base 2) form to represent decimal (base 10) numbers internally. Unfortunately, most decimal fractions can't be represented exactly in binary form, so small differences occur. 
    var x = 0.1 + 0.2;
console.log(x) // 0.30000000000000004
  • To avoid this problem, the fractions can be multiplied by a factor so they become "whole" numbers, operated on and than divided by the same factor.
    var x = (0.1 * 10 + 0.2 * 10) / 10;
console.log(x) // 0.3
  • Floating-point numbers are rounded up if longer than 16 digits after the floating pint ..
  • Integers (numbers without fractional parts or exponential notation) are rounded up if longer than 15 digits.
    var x = 999999999999999;
console.log(x); // 999999999999999

var y = 9999999999999999;
console.log(y); // 10000000000000000

boolean context

  • Non boolean values are treated as false (falsy) or true (truthy) in an "boolean context".
  • A "boolean context" is created when "logical operator" or "if condition" evaluate.
falsy value (evaluated to false in Boolean context) truthy value (evaluated to true in Boolean context)
0 "" or '' (empty string) null (when there is no value at all) undefined (when an declared value has no value) NaN everything that is not falsy is truthy

logical operators and short circuit

logical operators (evaluating true / truthy - false / falsy)
operator explanation short circuit
&& and Both conditions must evaluate to true for the entire condition to evaluate to true. If the first condition in an && condition evaluates to false, the second condition wont even be checked.
|| or Only one of the conditions must evaluate to true for the overall statement to evaluate to true. If the first condition in an || condition evaluates to true, the second condition wont even be checked.
! not (bang) Changes its boolean value to the opposite. (true becomes false)

short-circuit evaluation
example code explanation
let defaultName = username || 'Stranger';
  • The "or operator" checks from left to right.
  • Only the first value (username) on the left is checked.
  • truthy, only the left value (username) is assigned to the variable (defaultName).
  • falsy, only the right value ('Stranger') is assigned to the variable (defaultName).
 
let defaultName;
if (username) {
  defaultName = username;
} else {
  defaultName = 'Stranger';
}
 If the username would be undefined (falsy) then the else statement is executed. If the username has a value (truthy), then the if statement is executed.
