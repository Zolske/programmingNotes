loops

general rules for all loops

  1. Every loop has a iterator variable which can be set outside the loop or as part of its condition.
  2. The stopping condition is checked against the iterator variable, as long the condition is true the loop runs.
  3. The iterator variable needs to be updated so the "stopping condition" can become false and stop the loop!
  • The statement(s) within the loop are executed once for every loop (iteration).
  • The statement of the loop dose not need to be put in brackets if it is no more than one.
  • The iterator variable can be used within the loops statement.
  • Any global variable can be used inside the loop but they will be also changed by the loop.
  • i stands for iterator variable, but it could have any arbitrary name.
  • break; the parser exits the loop.
  • continue; the parser leaves out this one iteration and continues with the next iteration.

looping through an array (or string)

  • To loop through each element in an array, a for loop has to use the .length property of the array (or string) in its condition.
const vacationSpots = ['Bali', 'Paris', 'Tulum']; // instead of an array it could be also a string variable

for (let i = 0; i < vacationSpots.length; i++){
      console.log(`I would love to visit ${vacationSpots[i]}`);
} // I would love to visit Bali   // I would love to visit Paris   // I would love to visit Tulum

  console.log(value);

// 11
// 21
// 31

const tinasFollowers = ['Michael','Jack','Anne'];      //in two different arrays
const mutualFollowers = [];                            //same names to be saved in here

for (let i = 0; i < bobsFollowers.length; i++) {
  for (let z = 0; z < tinasFollowers.length; z++) {
    if (bobsFollowers[i] === tinasFollowers[z]) {      //if both elements are equal
        mutualFollowers.push(tinasFollowers[z]);       //pushes equal name into array
    };
  };
};
