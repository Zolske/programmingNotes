CSS Basic

Cascading Style Sheets is a style sheet language used for describing the presentation of a document written in a markup language such as HTML. This is done by "selecting" the html element and applying the css "property:values" to it.

implementing CSS

There are three methods to add css style to a html document.

explain: implementing CSS
  • 1. Inline Styles
    • <tag style=”PROPERTY:VALUE;PROPERTY2:VALUE2”>
    • is implemented directly by setting the style of the html element to the css "property:value"
    • overwrites all other css styles, except !important
  • 2. Embedded Styles
    • <head> <style> SELECTOR {PROPERTY:VALUE;} </style> </head>
    • is implemented in the head section of the html document, set between the style
    • overwrites external stylesheets, except !important
  • 3. External Styles
    • <head> <link rel="stylesheet" href="styles.css"> </head>
    • links to the external CSS file
  1. advantages of using an external stylesheet
  2. multiple web pages can be styled with one stylesheet
  3. better control over typography and page layouts. (more readable)
  4. style can be separate from structure. (separation of concerns)
  5. maintaining the site is simplified due to one-change-affects-all.

CSS syntax

How the CSS code has to be formed.

explain: CSS syntax
  • syntax
  • CSS is case sensitive
  • the declaration block needs to be wrapped with curly braces { }
  • each declaration in the block must end with a semicolon ;
  • properties need to be separated from there value with a colon :
  • naming classes and ids should be descriptive, separate a two word name with an hyphen - instead of camelCase
  • rule set
  • the selector selects the html element and applies the CSS rules to it
  • can have multiple selectors and declarations
  • if the specificity of rules which select the same tag is the same, than the last CSS rule overwrites the previous one

units of measurement

There are absolute- (fixed) and relative- (changes relative to ...) units to measure length.

explain: absolute units, pixel

Is fixed in relation to each other. The most common used unit is pixel (px).

  • a px (pixel) is ...
  • ... the basic unit of programmable colour on a computer display
  • ... the physical size of a pixel depends on the display screen resolution
    • if a pixel has a size of 1/96 of an inch (0.26 mm) than its screen resolution is 96 dpi (dots per inch) which used to be the standard but now screens have a higher screen resolution.
  • ... 16px is the default font size used by most browsers
  • ... used to set a fixed-size that will not change or be responsive to the screen size it is displayed on

explain: relative units, em, rem, %, vw & vh

Specifies a length that is relative to another object: root element, viewport, or parent. The result of the relative values depends on their relation to such objects. The result is calculated by the browser and converted into pixels. Relative units of measurement are responsive in their nature, meaning they are most useful when you need to take into account that your website will be viewed on many different screen sizes from large monitors to smartphones.

  • a em :
  • Is relative to the font-size of the element
    • (e.g. 2em means 2x the size of the current font)
  • If no font-size is defined anywhere in the CSS, the em unit will be equal to the browser's default font-size for the document (usually 16px)
  • Note: the font-size value is inherited from the parent element
  • a rem (stands for root em) :
  • Is relative to font-size of the root element, meaning the html
    • (e.g. if html is 16px than rem is also 16px)
  • If no font-size is defined anywhere in the CSS, the em unit will be equal to the browser's default font-size for the document (usually 16px)
  • Dose not depend on the values of the parent element.
  • a % percentage :
  • Relates to the size of its parent element.
    • (e.g. parent div 100px wide, than the child div with 50% width is 50px wide)
  • viewport units: vw (viewport width) & vh (viewport height) :
  • Sizing elements relative to the browser viewport.
  • The values are a percentage of the current browser viewport
    • (e.g. if viewport size is 100px than 10vw makes the element 10px wide)
    • (e.g. vh (100vh) is very useful to push the footer to the bottom of the page when there is not enough content)
