Z-index

The purpose of the z-index property is to control the vertical stacking of overlapping elements.

explain: z-index
  • z-indexVALUE
  • a higher z-index value appears to be physically closer to the viewer
  • values can be negative
  • with no z-index value set the elements stack in the same order as they appear in the HTML
    • the lowest in the hierarchy will appear closest to you
    • if element A is on top of element B, no child element of A can be higher in the stack than element B

overflow

Controls the content that falls out of the boundaries of the container.

explain: overflow

When the content is bigger than its container, than overflow occurs. The content "overflows" into the following elements which overlaps if there is not enough space in between, they are not pushed away.
Therefore it is generally a bad idea to give a static height to any element containing text.

  • overflowVALUE (all), overflow-yVALUE (vertical), overflow-xVALUE (horizontal)
  • visible (default)
    • is the default behavior, courses the content to overflow if its parent is to small
  • auto
    • a scrollbar will be shown only if it is needed
      (if content exceeds container)
  • scroll
    • always adds a scrollbar whether it is needed or not, for overflow both horizontally and vertically
  • hidden
    • hides the overflowing content, can not be accessed by the user
    • can be useful when dealing with dynamically generated content where an overflow will cause major layout issues

Floats

“Floats” let you put block-level elements side-by-side instead of on top of each other.

explain: float & clear

floats

Setting the float property courses the element to float to the side of it's patent element. Floated elements are removed from the normal flow of the page. They are ignored by the following element (they flow behind it) till the space behind the floated elements is filled out or the following element has the clear.

  • float left or right
  • floats the element respectively to the left or to the right side
example: float

no floats

  • no floats, default behavior

floatleft

  • the float of the div sidebar has been set to left
  • div sidebar is aligned to the left side
  • this curses the following div box to "flow" behind the div sidebar
  • Note: how the text flows around the floated div sidebar and not behind it

floatright

  • the float of the div box has been set to right
  • div box is aligned to the right side
  • this curses the following div footer to "flow" behind the div box and the div sidebar
  • if there would be more elements after div footer, than they would also "flow" behind the floated elements

clear

Stops the element from flowing behind a floated element.

  • clearVALUE
  • left or right
    • clears the float respectively on the left side and right side
  • both
    • clears both sides of the float
example: clear

clearleft

  • the clear of the div footer has been set to left
  • only the left side is "cleared" of the float (no flowing behind), the right side is still floating (flowing behind)

clearboth

  • the clear of the div footer has been set to both
  • both sids are "cleared" of the float (no flowing behind)
