html

HTML (HyperText Markup Language) defines the meaning and structure of web content. ... HTML uses "markup" to annotate text, images, and other content for display in a Web browser.
"Hypertext" refers to links that connect web pages to one another, either within a single website or between websites.

html structure

html element

Elements mark text or work as placeholder for images, videos, ...

  • element syntax
  • is not case sensitive (<img> == <IMG>) but the convention is lower case
  • elements are made up of either ...

    • a pair of start-end-tags (can hold "text-contend" between tags)
      html element
      start tag content end tag
      <h1 ATTRIBUTE> text content </h1>

    • just self-closing-tag (works as placeholder)
      html element
      self-closing tag
      <img ATTRIBUTE>
      or
      <img ATTRIBUTE/> in older versions, redundant since html5
  • ATTRIBUTES
  • types
    • element specific attributes, only certain elements can have (src target)
    • universal attributes, every element can have (id class title)
  • syntax
    • there is no limit of how many attributes an element can have
      (e.g. <p class="city" id="country">Berlin</p>)
    • there are two types of attributes ...
      • boolean attribute, either "true" or "false", no value, the presence of the attribute makes it "true" (e.g. required)
      • property-value pair, ATTRIBUTE="VALUE1 VALUE2" (can have multiple values separate by space)
        (e.g. class="city country")

  • class="VALUE1 VALUE2"
    • can be applied to any element, they are used by CSS and JavaScript to "select" an element
    • can have multiple values per attribute
    • the same class can be used on multiple elements per page

  • id="VALUE"
    • can be applied to any element, they are used by CSS and JavaScript to "select" an element
    • can have only one value per attribute but the element could have more ids
    • the same id can be used on only one element per page (ids have to be unique)

semantically neutral / structural elements

Have no semantic meaning, they are used to mark content, so they can be selected with the help of "classes" and "ids".

  • <span>
  • is a inline-level element
    • dose not start a new line and takes only the width and height of its content
    • is used to mark up words
    • can be nested within <div> but <div> can not be nested within <span>
  • <div>
  • is a block-level element
    • takes up the entire width of its parent container (element)
    • starts on a new line and pushes any following element into a new line below it
    • has no visible height if it is left empty

comments

Are ignored by the browser and not visible on the page. They help to explain the code to programmers.
