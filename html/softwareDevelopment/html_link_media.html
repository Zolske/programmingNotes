links and media content

links

a anchor

Link a html-element to another page or to another part of the same page.

explain links
a href link content /a
  • a anchor element
    • between a and /a is the content which will be displayed as the links element
      (can be any html element e.g. textContent, image, video, ...)

  • a href hypertext reference attribute
    • is required, contains the path to where the link leads
    • external, another page
      • to the top of an another page (href="https://somewhere.com")
      • to the element with an id of an another page (href="https://somewhere.com#ELEMENT_ID")
    • internal, another part of the same page
      • to the top of the same page (href="#")
      • to the element with an id of the same page (href="#ELEMENT_ID")
    • href mailto: opens the user's email program
      • href="mailto:someone@example.com" someone@example.com
    • href tel: opens the users calling app
      • href="tel:+123456789" +123456789
  • a target
    • target _blank opens the link in a new tab
      • target="_blank"
  • a download boolean attribute (no value)
    • downloads the in the href specified file download file
  • a rel
    • specifies the relationship between the current document and the linked document
    • search engines may use the information to determine how to index or display the page
      • relhelp link to a help document
      • relprev link to previous / relnext link to next
      • relnofollow links to an unendorsed document, like a paid link. ("nofollow" is used by Google, to specify that the Google search spider should not follow that link)
      • relnoopener prevents potential malicious behaviour from the destination document, which is useful for links to social media
      • relexternal indicates that the referenced document is not part of the same site as the current document

media content

img image

Embeds an image into the document.

explain img
img
  • self closing tag
  • embeds an image into the document
img src
  • is required, it contains the path to the image you want to embed
    • can be a image from an external website
    • or from within the websites file structure
img alt
  • holds a text description of the image
    • screen readers read this description out to their users so they know what the image means
    • is displayed on the page if the image can't be loaded (e.g. network errors, content blocking, or linkrot)
  • Setting this attribute to an empty string (alt="") indicates that this image is not a key part of the content (it’s decoration or a tracking pixel), and that non-visual browsers (screen readers) may omit it from rendering. Visual browsers will also hide the broken image icon if the alt is empty and the image failed to display.
img width and img height
  • controlling the size of the image (in pixel), if only of those attributes is set than the other one is scaled accordingly
  • if both are used than the browser can arrange the space for the image before it is loaded which mitigate content layout shifts
  • it is better practice to use CSS to control the image size

picture

Chooses different images depending on the width of the browser viewport.

used for ...
  • used for ...
  • ... art direction, Cropping or modifying images for different media conditions (for example, loading a simpler version of an image which has too many details, on smaller displays)
  • ... offering alternative image formats, for cases where certain formats are not supported
  • ... saving bandwidth and speeding page load times, loading the most appropriate image for the viewer's display.

explain picture
picture
  • contains the two self-closing source and img elements
  • the browser starts to compare the viewport-width with the value of the media from top to bottom
  • the source of the image changes depending on the viewport size
source media srcset
  • source media
    • media VALUE determines the minimum width when the in srcset specified image is used, can be wider but not smaller e.g. (min-width: 600px) from 600px up to ∞px
    • if viewport is smaller than the specified min-width than the next media is checked
  • source srcset
    • srcset VALUE specifics the path to the image which is used if media is "true"
img
  • is used if none of the media is "true" or if the browser dose not support picture

iframe

Embeds another web document within the current document. It displays a web page within a web page.

explain iframe
iframe
  • embeds another web document within the current document
iframe src
  • is required, define the address of the web page that will be embedded into the current page
iframe title
  • For accessibility reasons it is a good idea to give the iframe a title attribute for use with screen readers.
iframe width and iframe height
  • controlling the size of the embeded website (in pixel but % can be used), scrollbar is displayed for contend which dose not fit

audio

Embeds sound content into a web page.

explain audio
audio content if not loaded/audio
  • any html element (textContent, link, img, video, ...) between audio and /audio is displayed if the audio was not loaded (only if the file type exist but it is not supported by the browser, if file path is missing than the "controls" are greyed out)
audio src
  • is required, it contains the path to the audio you want to embed
audio controls boolean attribute (no value)
  • displays the audio control element on the page (play/pause, volume, seeking, playback speed, download)
audio loop boolean attribute (no value)
  • sets the audio file to start over each time it is finished
audio muted boolean attribute (no value)
  • the volume is initially set to zero
audio autoplay boolean attribute (no value)
  • will automatically play the audio on page load
  • bad practice, some browser will bock audio files being played automatically
  • it is recommended to set the audio to muted
    • audio track
    • is used to reference timed text tracks (or time-based data), for example, to handle displaying subtitles automatically
    audiosource src type/audio
    • source src
      • path to the audio file
    • source type
      • audio file type is either audio/mpeg or audio/ogg
        (MP3, WAV and OGG (OGG not in Safari) are supported by all mayor browsers)

    video

    Embeds video content into a web page.

    explain video
    video content if not loaded/video
    • any html element (textContent, link, img, video, ...) between video and /video is displayed if the video was not loaded (only if the file type exist but it is not supported by the browser, if file path is missing than the "controls" are greyed out)
    video src
    • is required, it contains the path to the video you want to embed
    video controls boolean attribute (no value)
    • displays the video control element on the page

    video loop boolean attribute (no value)
    • sets the video file to start over each time it is finished
    video muted boolean attribute (no value)
    • the volume is initially set to zero
    video autoplay boolean attribute (no value)
    • will automatically play the video on page load
    • bad practice, some browser will bock video files being played automatically
    • browsers may autoplay video if set to muted
    video width and video height
    • sets the size of the video
    video track
    • is used to reference timed text tracks (or time-based data), for example, to handle displaying subtitles automatically
    videosource src type/video
    • source src
      • path to the video file
    • source type
      • video file type is either video/webm, video/mp4 or video/ogg
        (MP4, WebM and OGG (OGG not in Safari) are supported by all mayor browsers)
