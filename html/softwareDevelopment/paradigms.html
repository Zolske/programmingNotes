Programming Paradigms

Programming paradigms are a way to classify programming languages based on their features. Languages can be classified into multiple paradigms.

declarative vs imperative paradigms

Most languages can be categorized in declarative- and imperative- paradigms.
The two are often used together (HTML and CSS together with JavaScript for websites).

an analogy

In an analogy, the waiter in an restaurant would be the declarative paradigm, he is only concerned with taking and bringing the right order not how the meal is made. The chefs in the kitchen would be the imperative paradigm, because they are concerned how the meal is made.
The state of our sandwich changes over time as data (the ingredients) are added and mixed. The initial state might be an empty plate the final state is a prepared sandwich ready to be eaten. This is all done according to the sequenced instructions (the recipe) and the mutable data which is the composition of the sandwich.

declarative imperative
focus "what happens" "how something happens"
how they work ... Removes as much as possible from the low-level details of how things happen. It tries to concern itself only with what happens. Work usually together with "imperative" programming languages. Explicitly tells the computer what to do and how to do it. Can work independently.
languages HTML, CSS, SQL Java, C#, C++, JavaScript, Python, Ruby
advantages & disadvantage
  • easy to use and a shallow learning curve (simple instructions)
  • often limited to a relatively small set of instructions
  • limited control over the results
  • much more control over what the programs can do
  • more complex in what they can achieve
  • more difficult to implement (syntax is stricter)
essential aspects
  • sequenced instructions:
  • instructions are represented as lines of code, which are executed one at a time from top to bottom
  • algorithm:
  • is the ability to apply logic and decision-making within instruction
  • mutable data:
  • data that can change over the duration of a program
  • is usually stored in variables

the evolution of imperative programming

  • "spaghetti" code:
  • "jumps" to other parts of the code through "Go To line X" executions
  • this made code difficult to read and follow ("spaghetti" code)

The following programming paradigms are in the group of imperative programming.

  • structured programming:
  • directs the code execution based on a true-or-false condition
  • if: executes or ignores code based on a condition being true or false
  • loop: repeats code as long as the condition is true

    • procedural programming:
    • (extension of structured programming)
    • function: groups related code together which can be used with different arguments multiple times

      • object oriented programming:
      • (extension of procedural programming)
      • groups related data and functions into objects
      • classes are used as blueprint for the creation of similar object
      • objects are instances of there class, they are created in the computers memory when the code of there class is executed

      • event (driven) programming:
      • (extension of procedural programming)
      • code is running in the background, waiting for an event to take place
      • once the event occurs, code that is listening for that event is triggered into action
