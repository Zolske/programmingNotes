forms

the form element

The form collects data from the user through input-control elements which are nested within *. The value of those elements can be set through the value. The name of the element is than paired with the value and sends through the form to its specified destination (action). The method of the form determines how the data is used.

SYNTAX form

data submission

Each input-control element must also have a name VALUE. The name VALUE is essential for processing the data the user has sent. It provides a key with which to pair the data in the database.

name attribute
  • The name VALUE is paired with the value VALUE from the same control-input element
  • Within the same form, the input-controls name VALUE has to be unique, or its value VALUE gets overwritten by its duplication.

input-control element

Input-control elements are used to gather information from the user which can be send with the form as name:value pair.

  • input-control elements
  • input
    • the type determines the look and properties of the input
    • If this attribute is not specified, the default type adopted is text.
  • textarea
    • a text area can hold an unlimited number of characters (e.g. comments or reviews)
  • select
    • a dropdown list with predefined options for the user to choose from, contains one or more option elements
  • datalist a reusable list
  • specifies a list of pre-defined options for one or multiple input
  • the datalistidVALUE needs to match the inputlistVALUE
  • example for datalist

    Note: The datalist tag is not supported in Safari 12.0 (or earlier).

the label element

Provides a name for the input-control element to make the purpose of it clear to the user.

label

Associating a label with an input-control element offers some major advantages:

  • The label text is not only visually associated with its corresponding text input; it is programmatically associated with it too. This means that, for example, a screen reader will read out the label when the user is focused on the form input, making it easier for an assistive technology user to understand what data should be entered.
  • When a user clicks or touches/taps a label, the browser passes the focus to its associated input (the resulting event is also raised for the input). That increased hit area for focusing the input provides an advantage to anyone trying to activate it — including those using a touch-screen device.
