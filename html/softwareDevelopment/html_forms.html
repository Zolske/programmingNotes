the form element

As the form element is for collecting data from the user, the form needs some important attributes to tell it where and how to send the data.

input types

How an input works varies depending on the value of its type attribute. If this attribute is not specified, the default type adopted is text.

input type-attributes
  • The input is a self-closing tag.
  • All input need to have a name attribute, or its value is not submitted with the form!
Type attribute Description Example extra attributes
text The default value. A single-line text field. Line-breaks are automatically removed from the input value. maxlength minlength pattern placeholder
number A control for entering a number. Displays a spinner and adds default validation when supported. Displays a numeric keypad in some devices with dynamic keypads. max min
password A single-line text field whose value is obscured. Will alert user if site is not secure. maxlength minlength pattern placeholder
email A field for editing an email address. Looks like a text input, but has validation parameters and relevant keyboard in supporting browsers and devices with dynamic keyboards. maxlength minlength multiple
checkbox A check box allowing single values to be selected/deselected. checked
radio A radio button, allowing a single value to be selected out of multiple choices with the same name value. checked
color A control for specifying a color; opening a color picker when active in supporting browsers.
file A control that lets the user select a file. Use the accept attribute to define the types of files that the control can select. accept multiple
image A graphical submit button. Displays an image defined by the src attribute. The alt attribute displays if the image src is missing.
range A control for entering a number whose exact value is not important. Displays as a range widget defaulting to the middle value. Used in conjunction min and max to define the range of acceptable values. max min
search A single-line text field for entering search strings. Line-breaks are automatically removed from the input value. May include a delete icon in supporting browsers that can be used to clear the field. Displays a search icon instead of enter key on some devices with dynamic keypads. maxlength minlength placeholder
tel A control for entering a telephone number. Displays a telephone keypad in some devices with dynamic keypads. maxlength minlength pattern placeholder
url A field for entering a URL. Looks like a text input, but has validation parameters and relevant keyboard in supporting browsers and devices with dynamic keyboards. maxlength minlength placeholder
hidden A control that is not displayed but whose value is submitted to the server. There is an example in the next column, but it's hidden!
time A control for entering a time value with no time zone. max min
date A control for entering a date (year, month, and day, with no time). Opens a date picker or numeric wheels for year, month, day when active in supporting browsers. max min
datetime-local A control for entering a date and time, with no time zone. Opens a date picker or numeric wheels for date- and time-components when active in supporting browsers. max min
week A control for entering a date consisting of a week-year number and a week number with no time zone. max min
month A control for entering a month and year, with no time zone. max min
button A push button with no default behavior displaying the value of the value attribute, empty by default.
reset A button that resets the contents of the form to default values. Not recommended.
submit A button that submits the form.

specific attributes for type-attribute
attribute type-attribute description example
name all type-attributes The value of the input is paired with the name attributes-value when the form is submitted. If there is no name-attribute, than no input-values can be submitted with the form.
Only one radio button in a same-named group of radio buttons can be checked at a time. Selecting any radio button in that group automatically deselects any currently-selected radio button in the same group. The value of that one checked radio button is sent along with the name if the form is submitted
Warning: Avoid giving form elements a name that corresponds to a built-in property of the form, since you would then override the predefined property or method with this reference to the corresponding input.

accept file Hint for expected file type in file upload controls, other files can be still chosen. 
<input type="file"
accept=".doc,.docx">
autofocus
boolean		 all type-attributes A Boolean attribute which, if present, indicates that the input should automatically have focus when the page has finished loading.
No more than one element in the document may have the autofocus attribute. If put on more than one element, the first one with the attribute receives focus.
Warning: Automatically focusing a form control can confuse visually-impaired people using screen-reading technology and people with cognitive impairments. When autofocus is assigned, screen-readers "teleport" their user to the form control without warning them beforehand.
 
<input type="text" autofocus>
checked
boolean		 radio checkbox Radio type * and checkbox type, are checked by default (when the page loads). It does not indicate whether this checkbox is currently checked, but if no changes are made by the user the value of the box with "checked" is submitted. 
<input  type="radio" name="A" checked>
<input type="radio" name="A">

<input type="checkbox" checked>
<input type="checkbox">
disabled
boolean		 all type-attributes Disabled inputs are typically rendered with a dimmer color or using some other form of indication that the field is not available for use.
Specifically, disabled inputs do not receive the click event, and disabled inputs are not submitted with the form.		 
<input type="text" disabled>
<input type="reset" disabled>
<input type="radio" disabled>
form all type-attributes The form attribute lets you place an input anywhere in the document but have it included with a form elsewhere in the document.
If this attribute isn't specified, the input element is associated with the nearest containing form, if any.
Note: An input can only be associated with one form.

max numeric values Defines the greatest value in the range of permitted values (integer). The form can not be submitted as long the number is greater, an error message is displayed. 
<form>
<input type="number" max="2">
<input type="submit">
</form>
min numeric values Defines the lowest value in the range of permitted values (integer). The form can not be submitted as long the number is lower, an error message is displayed. 
<form>
<input type="number" min="2">
<input type="submit">
</form>
maxlength text search url tel email password Defines the maximum number of characters (as UTF-16 code units) the user can enter into the field. This must be an integer value. 
<input type="text" maxlength="2">
minlength text search url tel email password Defines the minimum number of characters (as UTF-16 code units) the user can enter into the field. This must be an integer value. If there are less characters, an error message is displayed. 
<input type="text" minlength="2">
multiple
boolean		 email file The user can enter comma separated email addresses in the email widget or can choose more than one file with the file input.
pattern password text tel Is a regular expression that the input's value must match in order for the value to pass constraint validation. It must be a valid JavaScript regular expression, as used by the RegExp type.
