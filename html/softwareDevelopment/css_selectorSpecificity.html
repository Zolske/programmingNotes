CSS Selector & Specificity

Selectors & specificity

There are different selector which determine how a group or single html element is selected. Specificity determines which css-declaration applies to the element.

explain: selectors & specificity
Selector Specificity Syntax Description
!important
not a selector		 1,0,0,0,0 
SELECTOR {
    PROPERTY:VALUE !important ;
}
 Can only be overwritten with another !important which comes after or has combined with another selector a higher specificity.
Note: it is bad practice because it makes code difficult to read and debug.
inline style
not a selector		 0,1,0,0,0 
<HTML_ELEMENT style="PROPERTY:VALUE;">
 Overwrites any embedded-style and external-stylesheet, except if they have !important and the inline-style has not.
Note: html, css and JavaScript should be kept in separate files (separation of concerns)
embedded Styles
not a selector		 is not in specificity system 
<style>
    CSS_RULE_SET
</style>
 Overwrites any external-stylesheet, except if it has !important and the embedded-style has not.
Note: html, css and JavaScript should be kept in separate files (separation of concerns)
id selector
#		 0,0,1,0,0 
#ID_NAME {
    PROPERTY:VALUE;
}
 An ID (name) can be given only once per HTML document. But it can be used multiple times as selector in CSS.
class selector
.		 0,0,0,1,0 
.CLASS_NAME {
    PROPERTY:VALUE;
}
 The same class (name) can be used multiple times on HTML elements and as CSS selector.
attribute selector
ELEMENT[ATTRIBUTE]		 0,0,0,1,0 
ELEMENT[ATTRIBUTE] {
    PROPERTY:VALUE;
}
 Selects only the element which has the specified attribute.
pseudo-class
user action pseudo-classes These pseudo-classes require some interaction by the user in order for them to apply, such as holding a mouse pointer over an element.
pseudo-class Description Example
:hover SELECTOR:hover {}
Is triggered when the user hovers over the selected element with the cursor.
:active SELECTOR:active {}
The selected element becomes active only as long as it is clicked on with the (left) mouse-button.
:focus SELECTOR:focus {}
Is used to select element that have focus. Note: Is only allowed on elements that accept keyboard events or other user inputs.

tree-structural pseudo-classes These pseudo-classes relate to the location of an element within the document tree. more information
pseudo-class Description Syntax Example
:root In HTML, :root represents the <html> element and is identical to the selector html, except that its specificity is higher.

:root {}
Selects the HTML element.

 :root {background: coral;}
pseudo-class Description Example
:first-child SELECTOR1:first-child {}
  • every SELECTOR1 which is the first sibling Note: SELECTOR1 is still first if there are no SELECTOR1 siblings
  • can be paired with "combinator" to make selection more specific
:last-child SELECTOR1:last-child {}
  • every SELECTOR1 which is the last sibling Note: SELECTOR1 is still last if there are no SELECTOR1 siblings
  • can be paired with "combinator" to make selection more specific
:nth-child() SELECTOR1:nth-child(An+B)
  • Selects siblings based on the formula (An+B)
  • n
    • starts by 0 and increases with every iteration
      by 1 till there are no more children
  • An
    • the value of A is multiplied with n
  • n+B
    • the value of B is added to n
  • can be paired with "combinator" to make
    selection more specific
:nth-child(n+7){}

Represents the seventh and all following elements:
7=(0+7), 8=(1+7), 9=(1+7), ... etc.


:nth-child(3n+4){}

Represents elements:
4=((3x0)+4), 7=((3x1)+4), 10=((3x2)+4), 13=((3x3)+4), ... etc


:nth-child(2){}

Represents only the second element.


:nth-child(odd){}

Represents every element at an "odd" place.

:nth-child(2n+1)
is exactly the same as odd but the starting point can be changed with B


:nth-child(even){}

Represents every element at an "even" place.

:nth-child(2n)
is exactly the same as even but the starting point can be changed with A

: 
SELECTOR:PSEUDO_CLASS {
    PROPERTY:VALUE;
}
 Selects the element, class or id only if it is in the state specified by the pseudo-class.
element (type) selector
ELEMENT		 0,0,0,0,1 
ELEMENT {
    PROPERTY:VALUE;
}
 Selects all elements of the given type within a document.
pseudo-element

Adds extra textContent before or after the selected element.

pseudo-element Description
::after

::before

Both add textContent to the SELECTOR which can be styled. 

SELECTOR::before{
    content: "I am added before";
    color: red;
}
SELECTOR::after{
    content: "I am added after";
    color: red;
}

:: 
SELECTOR:PSEUDO_ELEMENT {
    PROPERTY:VALUE;
}
 The css-declaration applies only to the part of the element, class or id specified by the pseudo-element.
universal selector
*		 0 
* {
    PROPERTY:VALUE;
}
 Selects everything except pseudo-elements. Include pseudo-elements *::PSEUDO_ELEMENT (e.g. *::before *::after )
Note: it has the lowest specificity which means it is overwritten by any other css-rule and it cost the browser more performance.

Note: If the same element is selected by different selectors with the same specificity, than the last css-rule overwrites the previous matching css-property with its value.

combining selectors (combinator)

Combining two selector makes the them more specific.

explain: combinator
Selector Syntax Example
descendant
 		 
SELECTOR1 SELECTOR2 {
    PROPERTY:VALUE;
}
  • the descendant SELECTOR2 of every SELECTOR1
  • every SELECTOR2 which is nested within SELECTOR1
    • no matter how deep
    • no matter if other selectors are in between
  • Is applied to all descendant p of every div , no matter how deep
child
>		 
SELECTOR1 > SELECTOR2 {
    PROPERTY:VALUE;
}
  • Every SELECTOR2 which is a direct child of SELECTOR1.
  • only first generation
  • Every p which is a direct child of every div .
general sibling
~		 
SELECTOR1 ~ SELECTOR2 {
    PROPERTY:VALUE;
}
  • Every SELECTOR2 which is a sibling to SELECTOR1.
  • Without div the h2 and the p would be not related.
adjacent sibling
+		 
SELECTOR1 + SELECTOR2 {
    PROPERTY:VALUE;
}
  • Every SELECTOR2 which is sibling to SELECTOR1 and comes immediately after it.
  • The second p is not selected because it comes not after h2 .

chaining selectors together
Method Syntax Example
combining class selectors 
ELEMENT.CLASS_1.CLASS_2 {
    PROPERTY:VALUE;
}
  • no space between selectors
  • only first & only one element selector
  • increase specificity
only li with the class CLASS_1 and CLASS_2 is selected
Listing selectors
, 
SELECTOR_1, SELECTOR_2, SELECTOR_3 {
    PROPERTY:VALUE;
}
  • selectors (id, class, elements, ...)
  • every selector is selected
  • dose not increase specificity
